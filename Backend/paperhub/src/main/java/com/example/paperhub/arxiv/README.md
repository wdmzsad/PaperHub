# arXiv API 代理服务

## 功能说明

这是一个后端代理服务，用于解决前端直接访问 arXiv API 时的 CORS（跨域资源共享）问题。

## 工作原理

```
┌─────────────┐         ┌──────────────┐         ┌─────────────┐
│ Flutter Web │  ────>  │ 后端服务器   │  ────>  │  arXiv API  │
│  (浏览器)    │         │ (Spring Boot)│         │             │
└─────────────┘         └──────────────┘         └─────────────┘
     │                        │                        │
     │  GET /arxiv?id=xxx     │                        │
     │───────────────────────>│                        │
     │                        │ GET /api/query?id_list=xxx
     │                        │───────────────────────>│
     │                        │                        │
     │                        │      XML 响应          │
     │                        │<───────────────────────│
     │      XML 响应          │                        │
     │<───────────────────────│                        │
```

## API 端点

### GET /arxiv

获取 arXiv 文献元数据

**请求参数：**
- `id` (必需): arXiv ID，例如 `1512.03385` 或 `2301.12345`

**示例请求：**
```bash
GET http://localhost:8080/arxiv?id=1512.03385
```

**响应：**
- 成功 (200): 返回 XML 格式的文献元数据
- 错误 (400/404/500): 返回错误消息

## 配置说明

### 后端配置

1. **控制器位置：**
   - `com.example.paperhub.arxiv.ArxivController`

2. **依赖：**
   - Spring Boot Web
   - RestTemplate (已配置在 `RestTemplateConfig`)

3. **CORS 配置：**
   - 已启用跨域访问 (`@CrossOrigin(origins = "*")`)

### 前端配置

前端服务 (`Frontend/lib/services/arxiv_service.dart`) 已配置为：
- 使用后端代理：`http://localhost:8080/arxiv`
- 如果后端地址不同，请修改 `_proxyBaseUrl` 常量

## 优势

1. **解决 CORS 问题**：浏览器无法直接访问 arXiv API，但服务器可以
2. **统一错误处理**：在后端统一处理错误，返回友好的错误消息
3. **安全性**：可以添加认证、限流等安全措施
4. **缓存**：可以在后端添加缓存机制，减少对 arXiv API 的请求

## 故障排查

### 如果返回 404

1. 确认后端服务正在运行
2. 检查端点路径是否正确：`/arxiv`（不是 `/api/arxiv`）
3. 查看后端日志，确认请求是否到达

### 如果返回网络错误

1. 确认后端服务器能访问 `export.arxiv.org`
2. 检查防火墙设置
3. 查看后端日志中的详细错误信息

### 如果前端显示 CORS 错误

1. 确认前端配置使用后端代理（`_useProxy = true`）
2. 确认后端地址正确
3. 检查后端 CORS 配置

## 扩展功能
