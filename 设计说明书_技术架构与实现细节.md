# PaperHub 设计说明书问题回答（技术架构与实现细节）

> 说明：以下内容基于当前项目源码（`Frontend` + `Backend/paperhub`）整理，尽量只写代码中可确认的信息。

---

## 1. 技术架构

### 1.1 整体架构
- 形态：以**移动端/多端 App**为主（Flutter 一套代码多端运行）
- 架构模式：**前后端分离**
- 访问模式：前端通过 HTTP/JSON 调用后端 REST API，部分场景通过 WebSocket 实时推送
- 从部署角度看属于典型 **B/S + App API 架构**（客户端访问后端服务）

### 1.2 前端技术栈
- 框架：**Flutter（Dart）**
- 工程特征：包含 Android / iOS / Web / Desktop 目录，说明采用 Flutter 跨平台方案
- 关键模块：`ApiService`（HTTP 封装 + Token 刷新）、聊天与通知 WebSocket 客户端、页面与状态管理

### 1.3 后端技术栈
- 语言：**Java**
- 框架：**Spring Boot**
- 主要组件：
  - Spring Web（REST 控制器）
  - Spring Data JPA（数据访问）
  - Spring Security（当前安全策略放开）
  - JavaMailSender（邮件）
  - WebSocket（原生 Spring WebSocket，非 STOMP）
  - Redis（聊天缓存、实时消息相关）

### 1.4 数据库
- 主数据库：**MySQL**（`application.properties` 中配置 MySQL JDBC）
- ORM：JPA/Hibernate

### 1.5 服务器/部署
- 代码中存在云端地址（如 `124.70.87.106:8080`），README 有云服务器部署步骤（Nginx + Java 进程）
- CI/CD：存在 `.gitlab-ci.yml`，使用 GitLab Runner 构建前后端
- 文件存储：使用华为云 OBS（对象存储）
- 结论：当前为**云服务器部署 + 对象存储 + CI 构建**，非 Docker 化主流程（源码中未见 Dockerfile/compose）

### 1.6 选择该技术栈的原因（简述）
- Flutter 可一套代码覆盖多端，适合课程项目快速迭代与统一体验。
- Spring Boot + MySQL + JPA 成熟稳定、上手快，适合中小型社区类业务。
- OBS、SMTP、WebSocket、Redis 组合能覆盖文件、通知、实时通信和性能优化需求。

## 2. 后端模块划分（按包）

> 后端主要位于 `com.example.paperhub.*`

- `auth`：注册、登录、邮箱验证、重置密码、角色与用户状态
- `jwt`：JWT Access/Refresh Token 生成与校验
- `user`：个人资料、隐私设置、头像/背景图上传、用户搜索
- `post`：帖子发布/编辑/删除、分区与标签、推荐流、关注流、搜索、草稿
- `comment`：评论/回复/编辑/删除、评论树结构
- `like`：帖子点赞、评论点赞
- `favorite`：帖子收藏
- `follow`：关注/取关、粉丝/关注/互关列表
- `chat`：私信会话、消息、已读状态、消息缓存
- `notification`：点赞/评论/@/关注等通知查询与已读管理
- `history`：浏览历史、搜索历史
- `hot`：热搜榜计算与查询
- `arxiv`：arXiv API 代理（解决前端 CORS）
- `report`：帖子举报流程、作者整改提审、管理员审核流
- `admin`：后台用户管理、帖子管理、举报管理、公告管理、管理员权限管理
- `websocket`：帖子互动、通知、私信实时推送通道
- `config`：安全、异常、OBS 等系统配置
- `notify`：邮件发送服务（验证码、重置密码）

## 3. 前后端通信方式

### 3.1 协议
- 主通信：**HTTP/HTTPS + JSON**
- 实时通信：**WebSocket**

### 3.2 API Base URL 格式
- 前端 `AppEnv`：
  - 本地默认：`http://localhost:8080`
  - 云端默认：`http://124.70.87.106:8080`
- 接口路径风格并不统一，常见前缀包括：
  - `/auth/*`
  - `/users/*`
  - `/posts/*`
  - `/notifications/*`
  - `/api/*`（如举报、聊天、部分管理接口）

### 3.3 Token 认证
- 已实现 JWT：
  - 登录返回 `token + refreshToken`
  - 前端统一处理 401 后自动调用 `/auth/refresh` 重试
- 但当前安全配置是 `permitAll` 且未挂载 JWT 过滤器，属于开发期放开状态；设计上是 Token 认证，运行上当前“弱约束”。

## 4. 关键功能实现算法/流程

### 4.a 推荐算法（首页个性化推荐）

#### 是否基于用户行为
- 是。综合使用：
  - 研究方向
  - 浏览历史
  - 收藏历史
  - 自己发帖标签偏好
  - 最近搜索关键词
  - 帖子热度与时间新鲜度

#### 是否使用大模型
- **未使用大模型**  
- 属于规则/特征加权的排序算法（标签匹配 + 热度 + 时间 + 行为偏好）

#### 完整流程
1. 用户请求 `/posts/recommendations`  
2. 若未登录：降级为普通时间排序（近似 `/posts`）。  
3. 若已登录：`RecommendationService` 构建用户兴趣画像：
   - 研究方向（`researchDirections`）
   - 最近搜索关键词
   - 最近浏览帖子标签
   - 最近收藏帖子标签
   - 自己发帖标签
4. 取最近最多 200 条 `NORMAL` 帖子作为候选集。  
5. 对每条候选帖计算分数（线性加权）：
   - 研究方向匹配
   - 收藏兴趣匹配
   - 发帖兴趣匹配
   - 浏览兴趣匹配
   - 搜索关键词匹配
   - 时间新鲜度归一化
   - 热度归一化（likes/comments/views）  
6. 对“自己发过/已看过”的帖子做惩罚降权。  
7. 按总分降序排序，内存分页后返回。

---

### 4.b 热榜机制

#### 计算公式
- `HotSearchService` 注释与实现给出：
  - 热度分数 = 搜索次数 × 用户因子 × 时间衰减因子
  - 用户因子：`log10(独立用户数+1) * 2.0`
  - 时间衰减权重：
    - 1h: 1.0
    - 1-6h: 0.7
    - 6-24h: 0.4
    - 24h+: 0.1
- 增长率用于打标：
  - 新上榜高增长标“新”
  - 持续高增长标“热”

#### 更新频率
- 定时任务：`@Scheduled(cron = "0 */10 * * * *")`
- 即：**每 10 分钟更新一次**
- 统计窗口：最近 24 小时

---

### 4.c 搜索功能

- 实现方式：主要是 **MySQL + JPA Query（LIKE）**，未见 Elasticsearch
- 帖子搜索范围：
  - 标题 `title`
  - 正文 `content`
  - 标签 `tags`
- 支持：
  - `type=keyword`（关键词搜索）
  - `type=tag`（标签搜索）
  - `sort=hot|new`（热度或最新）
- 用户搜索：按用户名（`name`）模糊检索并结合热度评分排序

---

### 4.d 私信实时通讯

- 是否实时：**是**
- 技术：**WebSocket（原生 Spring WebSocket） + Redis 缓存 + DB 持久化**
- 连接端点：
  - 私信：`/ws/chat/{userId}`
  - 互动/通知相关还有：`/ws/posts/{postId}`、`/ws/notifications/{userId}`、`/ws/admin`
- 消息流程：HTTP 发消息入库 -> 推送给会话参与者 -> Redis 缓存最近消息 -> 客户端实时接收

---

### 4.e arXiv 信息抓取

- 不是爬虫，采用 **调用 arXiv 官方 API**：
  - `https://export.arxiv.org/api/query?id_list=...`
- 流程：
1. 前端输入 arXiv ID 或链接（支持多种格式解析）。  
2. 前端调用后端代理接口：`GET /arxiv?id=...`。  
3. 后端 `ArxivController` 转发请求至 arXiv API。  
4. 返回 XML 给前端。  
5. 前端 `ArxivService` 解析 XML，提取标题、作者、摘要、日期、分类、doi、journal 等。  
6. 发布笔记时把 arXiv 元数据写入帖子字段。

---

### 4.f 邮箱验证码

- 用途：注册验证、找回密码
- 发送方式：**SMTP（Spring JavaMailSender）**
- 邮件服务：配置为阿里云邮箱 SMTP（`spring.mail.host=smtpdm.aliyun.com`）
- 逻辑：
  - 注册验证码 6 位，5 分钟有效
  - 重置密码验证码 6 位，10 分钟有效

## 5. 关键数据结构（核心实体字段）

> 仅列主要字段（类名按代码名；“Note”对应 `Post`）

### 5.1 User（用户）
- `id`：用户ID
- `email`：邮箱（登录标识）
- `passwordHash`：密码哈希
- `name`：昵称
- `avatar`：头像URL
- `affiliation`：机构
- `bio`：简介
- `role`：角色（USER/ADMIN/SUPER_ADMIN）
- `status`：状态（NORMAL/BANNED/MUTE/AUDIT）
- `muteUntil`：禁言截止时间
- `researchDirections`：研究方向
- `profileBackground`：个人主页背景图
- `hideFollowing/hideFollowers/publicFavorites`：隐私配置
- `verified/verifyCode/verifyExpiry`：注册验证状态与验证码
- `resetCode/resetExpiry`：重置密码验证码

### 5.2 Note（笔记，对应 Post）
- `id`：帖子ID
- `title`：标题
- `content`：正文
- `media`：媒体列表
- `tags`：二级标签列表
- `mainDiscipline`：主分区
- `author`：作者
- `doi/journal/year`：论文元数据
- `externalLinks`：外链
- `arxivId/arxivAuthors/arxivPublishedDate/arxivCategories`：arXiv 元数据
- `references`：引用帖子ID列表
- `likesCount/commentsCount/favoriteCount/viewsCount`：统计计数
- `status`：帖子状态（NORMAL/DRAFT/AUDIT/REMOVED）
- `hiddenReason/updatedByAdmin/visibleToAuthor`：审核与下架相关
- `createdAt/updatedAt`：时间戳

### 5.3 Comment（评论）
- `id`：评论ID
- `post`：所属帖子
- `author`：评论作者
- `content`：评论内容
- `parent`：父评论（楼中楼）
- `replyTo`：被回复用户
- `likesCount`：评论点赞数
- `mentionIds`：@用户ID集合（字符串存储）
- `createdAt/updatedAt`：时间戳

### 5.4 Like（点赞记录）
- 帖子点赞 `PostLike`：
  - `id`
  - `post`
  - `user`
  - `createdAt`
- 评论点赞 `CommentLike`：
  - `id`
  - `comment`
  - `user`
  - `createdAt`

### 5.5 Favorite/Collection（收藏）
- `FavoritePost`
  - `id`
  - `user`
  - `post`
  - `createdAt`

### 5.6 Follow（关注关系）
- `UserFollow`
  - `id`
  - `follower`（谁关注）
  - `following`（被关注者）
  - `createdAt`

### 5.7 Message（私信）
- `id`
- `conversation`
- `senderId`
- `type`（TEXT/IMAGE/VOICE等）
- `content`
- `mediaUrls`
- `fileUrl/fileName/fileSize`
- `createdAt`

### 5.8 Report（举报）
- 帖子举报 `ReportPost`：
  - `id`
  - `reporter`
  - `post`
  - `description`
  - `reportTime`
  - `status`
  - `admin`
  - `handleTime`
  - `handleResult`
  - `postStatusAfter`
- 综合举报 `AdminReport`（用户/帖子/评论）：
  - `id`
  - `reporter`
  - `reportedUser`
  - `post`
  - `comment`
  - `targetType`
  - `reason`
  - `status`
  - `resolution`
  - `handledBy`
  - `createdAt/updatedAt`

### 5.9 Notification（通知）
- `id`
- `recipient`（接收者）
- `actor`（触发者）
- `type`（通知类型）
- `post`（可空）
- `comment`（可空）
- `read`（是否已读）
- `createdAt`

## 6. 第三方服务 / 外部接口

### 6.1 已使用的外部服务
- **arXiv API**：论文元数据查询
- **SMTP 邮件服务（阿里云邮箱）**：注册/重置验证码邮件
- **华为云 OBS**：头像、帖子附件、聊天文件/语音等对象存储
- **Redis**：消息缓存、会话实时相关
- **WebSocket 服务（后端自建）**：私信/通知/互动实时推送

### 6.2 文件存储方式
- 非本地磁盘主存储，核心上传文件走**云对象存储（华为云 OBS）**
- 后端返回可访问 URL，前端直接展示/下载

### 6.3 大模型/AI接口
- 当前代码中**未发现大模型 API 调用**
